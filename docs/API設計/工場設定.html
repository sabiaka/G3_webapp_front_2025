対象ページ: 工場設定ページ

このドキュメントは、「工場設定ページ」で利用するマスターデータ（権限、生産ライン）を管理するためのAPI仕様を定義します。

0. DB設計

このAPIは以下のマスターテーブルを操作します。

roles（権限マスタ）

-- PostgreSQL 例: 管理者かどうかを boolean で管理（true=管理者, false=一般従業員）
CREATE TABLE roles (
        role_id SERIAL PRIMARY KEY,
        role_name VARCHAR(50) UNIQUE NOT NULL,
        is_admin BOOLEAN NOT NULL DEFAULT FALSE
);

production_lines（ラインマスタ）

CREATE TABLE production_lines (
    line_id SERIAL PRIMARY KEY,
    line_name VARCHAR(100) UNIQUE NOT NULL
);

1. 権限種別 (Roles) API

1.1 権限一覧取得

GET /api/roles

登録されているすべてのロール（ロール名 + is_admin）を一覧で取得します。is_admin=true は「管理者」、false は「一般従業員」を表します。

成功レスポンス (200 OK)

[
    { "role_id": 1, "role_name": "管理画面運用", "is_admin": true },
    { "role_id": 2, "role_name": "閲覧のみ", "is_admin": false },
    { "role_id": 3, "role_name": "検査業務", "is_admin": false }
]

1.2 権限追加

POST /api/roles

新しいロールを登録します（ロール名は自由入力、is_admin は true/false の二択）。

リクエストボディ

{
    "role_name": "閲覧のみ",
    "is_admin": false // 省略時は false
}

成功レスポンス (201 Created)

{
    "role_id": 4,
    "role_name": "閲覧のみ",
    "is_admin": false
}

1.4 権限更新

PUT /api/roles/{id}

指定ロールの「ロール名」または「is_admin（管理者かどうか）」を更新します。

リクエストボディ（どちらか、または両方）

{
    "role_name": "検査リーダー",
    "is_admin": true // boolean
}

成功レスポンス (200 OK)

{
    "role_id": 3,
    "role_name": "検査リーダー",
    "is_admin": true
}

バリデーション:

  • role_name は重複不可（409 Conflict）
  • is_admin は boolean 以外（null/文字列など）は 400 Bad Request
  • 対象が存在しない場合は 404 Not Found

1.5 is_admin トグル

PATCH /api/roles/{id}/toggle

is_admin を true と false の間でトグルします。

成功レスポンス (200 OK)

{
    "role_id": 2,
    "role_name": "閲覧のみ",
    "is_admin": true
}

備考: UIの「押すたびに変わる」要求に対応する軽量API。PUTでも同様の更新は可能。

1.3 権限削除

DELETE /api/roles/{id}

指定されたIDのロールを削除します。

成功レスポンス (204 No Content)

削除が成功した場合、ボディには何も返されません。

注意:

  • ユーザーが紐づく等で削除不可の場合は 409 Conflict を返す（例: 外部キー制約違反）
  • 必要に応じてソフトデリート（is_deleted）で運用

2. 工場ライン (Production Lines) API

2.1 ライン一覧取得

GET /api/lines

登録されているすべての工場ラインを一覧で取得します。

成功レスポンス (200 OK)

[
    { "line_id": 1, "line_name": "Aライン (組立)" },
    { "line_id": 2, "line_name": "Bライン (塗装)" },
    { "line_id": 3, "line_name": "検査ライン" }
]

2.2 ライン追加

POST /api/lines

新しい工場ラインを登録します。

リクエストボディ

{
    "line_name": "Cライン (梱包)"
}

成功レスポンス (201 Created)

{
    "line_id": 4,
    "line_name": "Cライン (梱包)"
}

2.3 ライン名更新

PUT /api/lines/{id}

指定されたIDの工場ラインの名前を更新します。

リクエストボディ

{
    "line_name": "Dライン (最終検査)"
}

成功レスポンス (200 OK)

{
    "line_id": 2,
    "line_name": "Dライン (最終検査)"
}

バリデーション:

  • line_name は重複不可（409 Conflict）
  • 対象が存在しない場合は 404 Not Found

2.4 ライン削除

DELETE /api/lines/{id}

指定されたIDの工場ラインを削除します。

成功レスポンス (204 No Content)

削除が成功した場合、ボディには何も返されません。
