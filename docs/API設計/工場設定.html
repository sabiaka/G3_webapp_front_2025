対象ページ: 工場設定ページ

このドキュメントは、「工場設定ページ」で利用するマスターデータ（権限、生産ライン）を管理するためのAPI仕様を定義します。

0. DB設計

このAPIは以下のマスターテーブルを操作します。

roles（権限マスタ）

CREATE TABLE roles (
    role_id SERIAL PRIMARY KEY,
    role_name VARCHAR(50) UNIQUE NOT NULL
);

production_lines（ラインマスタ）

CREATE TABLE production_lines (
    line_id SERIAL PRIMARY KEY,
    line_name VARCHAR(100) UNIQUE NOT NULL
);

1. 権限種別 (Roles) API

1.1 権限一覧取得

GET /api/roles

登録されているすべての権限種別を一覧で取得します。

成功レスポンス (200 OK)

[
    { "role_id": 1, "role_name": "管理者" },
    { "role_id": 2, "role_name": "一般" },
    { "role_id": 3, "role_name": "検査員" }
]

1.2 権限追加

POST /api/roles

新しい権限種別を登録します。

リクエストボディ

{
    "role_name": "閲覧者"
}

成功レスポンス (201 Created)

{
    "role_id": 4,
    "role_name": "閲覧者"
}

1.3 権限削除

DELETE /api/roles/{id}

指定されたIDの権限種別を削除します。

成功レスポンス (204 No Content)

削除が成功した場合、ボディには何も返されません。

2. 工場ライン (Production Lines) API

2.1 ライン一覧取得

GET /api/lines

登録されているすべての工場ラインを一覧で取得します。

成功レスポンス (200 OK)

[
    { "line_id": 1, "line_name": "Aライン (組立)" },
    { "line_id": 2, "line_name": "Bライン (塗装)" },
    { "line_id": 3, "line_name": "検査ライン" }
]

2.2 ライン追加

POST /api/lines

新しい工場ラインを登録します。

リクエストボディ

{
    "line_name": "Cライン (梱包)"
}

成功レスポンス (201 Created)

{
    "line_id": 4,
    "line_name": "Cライン (梱包)"
}

2.3 ライン削除

DELETE /api/lines/{id}

指定されたIDの工場ラインを削除します。

成功レスポンス (204 No Content)

削除が成功した場合、ボディには何も返されません。
