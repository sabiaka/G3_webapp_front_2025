対象: Jetsonからのファイルの送信

このドキュメントは、PLCや画像検査プログラムからの信号をDBに格納するためのAPI仕様を定義します。

1. DB設計

このページで扱うデータは主に「検査結果」と「生産カウンタ」です。以下は対応するテーブル定義です。

inspection_results（検査結果）

CREATE TABLE inspection_results (
    inspection_id SERIAL PRIMARY KEY,                                   -- 自動採番
    inspection_camera_id VARCHAR(50) NOT NULL,                          -- カメラ名
    inspection_product_id VARCHAR(50) NOT NULL,                         -- 製品名
    inspection_result VARCHAR(50) NULL,                                 -- 検査結果（良品、不良など）
    inspection_reason TEXT NULL,                                        -- 問題点詳細
    inspection_image_path TEXT NOT NULL,                                -- 検査画像パス
    inspection_captured_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,       -- 撮影日時
    inspection_inspected_time TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP  -- 検査日時
);

POST /api/ingress/inspection で受け取る metadata と画像は、本テーブルに保存します。

machine_production（ホックリング残数・生産数）

CREATE TABLE machine_production (
    machine_prod_id SERIAL PRIMARY KEY,                             -- 生産データID（自動採番）
    machine_prod_captured_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP, -- 記録日時
    machine_prod_hook_remaining INT NOT NULL,                       -- ホックリング残数
    machine_prod_count INT NOT NULL                                 -- 生産数
);

POST /api/ingress/plc で受け取るカウント情報の格納先です（必要に応じて machine_id 等の列追加をご検討ください）。

3. PLC情報送信 (機械 → サーバー)

POST /api/ingress/plc 優先度: A

生産機械 (PLC) が、製品を1個作るたびに「できたよー！」ってサーバーに報告するためのAPI。実績数がリアルタイムで増えてくのは、こいつのおかげ！

リクエストボディ

{
    "machine_id": "press-machine-01",
    "product_id": "A-102-001",
    "count": 1,
    "timestamp": "2025-08-04T11:20:35Z"
}

成功レスポンス (202 Accepted)

データを受け取ったら、サーバーはすぐ「りょ！」って返事して、裏側でゆっくりデータを処理する。だから`202 Accepted`がイケてる。返事の中身はこんな感じのJSONだよ！

{
    "message": "データを受け付けました。処理を開始します。"
}

4. 画像と検査データを送信 (撮影/検査Jetson → サーバー)

POST /api/ingress/inspection 超重要！

撮影担当Jetsonと検査担当Jetsonが、どっちも使う最強API！ サーバーは送られてきた product_id がもう存在するかチェックして、なかったら新規作成、あったらデータと画像をまるっと上書きする感じ！

metadata パートのJSON例

タイムスタンプ改善！

いつ撮って、いつ検査したか分かるように、captured_at と inspected_at に分けたよ！これで分かりみUP！

① 撮影Jetsonが送る初期データ

{
    "camera_id": "inspection-camera-01",
    "product_id": "A-102-002",
    "result": null,
    "reason": null,
    "captured_at": "2025-09-12T05:09:04Z",
    "inspected_at": null
}

② 検査Jetsonが送る最終データ

{
    "camera_id": "inspection-camera-01",
    "product_id": "A-102-002",
    "result": "fail",
    "reason": "傷",
    "captured_at": "2025-09-12T05:09:04Z",
    "inspected_at": "2025-09-12T05:15:20Z"
}

Pythonでの送信コード例

検査Jetsonは、GETで受け取ったデータを元にmetadata作って、枠付けした画像をセットして送ればOK！


import requests
import json
import datetime

# --- どっちのJetsonでも共通 ---
url = "http://your-server.com/api/ingress/inspection"

# --- ② 検査Jetsonが、検査結果を送る場合の例 ---
# GET APIから受け取ったcaptured_atはそのまま使うのがイケてる
captured_timestamp = "2025-09-12T05:09:04Z" # GETレスポンスから取得したと仮定

metadata_for_update = {
  "camera_id": "inspection-camera-01",
  "product_id": "A-102-002",
  "result": "fail", # "pass" or "fail"
  "reason": "傷", # NGの理由
  "captured_at": captured_timestamp, # 撮影時間はそのまま
  # 検査した"今"の時間をセット！
  "inspected_at": datetime.datetime.now(datetime.timezone.utc).isoformat()
}

files = {
    'metadata': (None, json.dumps(metadata_for_update), 'application/json'),
    # 枠付けとかした、加工後の画像をセットしてね！
    'image_file': ('inspected_image.jpg', open('/path/to/processed_image.jpg', 'rb'), 'image/jpeg')
}

response = requests.post(url, files=files)

print(f"ステータスコード: {response.status_code}")
print("レスポンス:", response.json())

成功レスポンス (202 Accepted)

データと画像をちゃんと受け取れたら、「OK、受け取った！」って返事する。これで通信完了！

{ "message": "データを受け付け、処理を開始します。" }

5. 画像のアクセス方法 (超シンプル版)

GET /imageDB/{product_id}.jpg

Node.jsのExpressとかでpublicフォルダを公開設定にしとけば、URLを直で叩くだけで画像が見れちゃう。マジ卍で楽ちん！

アクセスURLの例

http://your-server.com/imageDB/A-102-002.jpg

6. 未検査の画像とデータをゲトるAPI (検査Jetson ← サーバー)

GET /api/inspections/uninspected 継続！

検査担当のJetsonが「次どれ検査すればいい？」って聞くためのAPI。サーバーに溜まってる未検査データの中から、一番古いやつを1個だけ選んで、そのメタデータと画像をbase64形式で返してくれる！

成功レスポンス (200 OK)

うまくいくと、次に検査すべきアイテムのデータがJSONで返ってくる。画像はbase64エンコードされてるから、デコードすればすぐに使えるよ！

{
    "metadata": {
        "camera_id": "inspection-camera-01",
        "product_id": "A-102-005",
        "captured_at": "2025-09-13T10:00:00Z"
    },
    "image_base64": "iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAQAAAC1HAwCAAAAC0lEQVR42mNkYAAAAAYAAjCB0C8AAAAASUVORK5CYII="
}

レスポンス (204 No Content)

もし未検査のアイテムが1個もなかったら、サーバーは「今はないよー」って感じで、中身が空っぽの 204 No Content を返すよ！
