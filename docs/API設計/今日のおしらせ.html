今日のお知らせAPI 設計書

対象ページ: ダッシュボード（トップページ）

このドキュメントは、ダッシュボードの「今日のお知らせ」機能のAPI仕様を定義します。

これベースで開発ヨロ！

0. DB設計

お知らせ機能は、単一のレコード（行）だけを持つシンプルなテーブルで管理します。

today_message（お知らせテーブル）

CREATE TABLE today_message (
    message_id INT PRIMARY KEY DEFAULT 1,       -- 常にID=1のレコードだけを使う
    content TEXT,                               -- お知らせの本文
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP -- 最終更新日時
);

※フロントエンド側は、このテーブルの message_id = 1 のレコードのみを対象にGETとPUTを行います。

※サーバー起動時やAPI側で、ID=1のレコードが存在しない場合のみ作成（INSERT）する処理を推奨します。

1. お知らせ取得

GET /api/message 優先度: S

現在設定されている「今日のお知らせ」の内容をGETするAPI。

クエリパラメータ

なし。

成功レスポンス (200 OK)

{
    "content": "qwer",
    "updated_at": "2025-10-27T15:00:00+09:00"
}

2. お知らせ更新

PUT /api/message 優先度: S

お知らせの内容を更新（上書き）するAPI。リソースが「ID=1」で固定されているため、PUTでまるっと更新するのがシンプル。

リクエストボディ

{
    "content": "新しいお知らせのテキストです。"
}

成功レスポンス (200 OK)

更新が成功したら、最新のデータが返ってくるよ。

{
    "content": "新しいお知らせのテキストです。",
    "updated_at": "2025-10-27T15:05:00+09:00"
}
