画像検査API 設計書（カラム名修正版）

← トップページに戻る

対象ページ: 画像検査ステータス

このドキュメントは、「画像検査ステータス」ページの最新フロント実装に合わせたAPI/DB設計を定義します。（※DBカラム名 4k_seq を c4k_seq に修正済み）

改訂概要

  • 重要修正: DBカラム名 4k_seq はSQL構文エラーとなるため、c4k_seq に変更しました。APIのJSONキー（4k_seq）とのマッピングに注意してください。
  • リアルタイム更新: WebSocket → 軽量GETポーリングに変更。
  • DBスキーマ: ロット＋カメラショットの2層構造へ再設計。
  • ショット詳細: 4K / FHD ごとにエンドポイントを分割。

1. データベーススキーマ（確定版）

PostgreSQLを想定。4k_seq は数字始まりでエラーになるため、DB上では c4k_seq とします。


-- ロット（製品ID等に相当）
CREATE TABLE inspection_lots (
    id BIGSERIAL PRIMARY KEY,                                       -- ロット内部ID
    lot_id VARCHAR(64) NOT NULL,                                    -- ロットID（例: 'MAT-10001'）
    section_code VARCHAR(32) NOT NULL,                              -- セクション（例: 'spring'）
    captured_at TIMESTAMP WITH TIME ZONE NOT NULL,                  -- 検査開始時刻
    UNIQUE (lot_id)
);

-- カメラショット
CREATE TABLE inspection_camera_shots (
    id BIGSERIAL PRIMARY KEY,
    lot_id_ref BIGINT NOT NULL REFERENCES inspection_lots(id) ON DELETE CASCADE,
    camera_id VARCHAR(64) NOT NULL,
    
    -- ▼ 修正箇所 ▼
    c4k_seq VARCHAR(32) NULL,                                       -- 4KシーケンスID（JSONキー "4k_seq" とマッピング）
    -- ▲ 修正箇所 ▲
    
    fhd_seq VARCHAR(32) NULL,                                       -- FHDシーケンスID
    status VARCHAR(16) NOT NULL CHECK (status IN ('PASS','FAIL','MISSING')),
    details TEXT NULL,
    image_path TEXT NULL,
    shot_seq INT NULL,                                              -- 旧互換用（NULL許容）
    captured_at TIMESTAMP WITH TIME ZONE NULL
);

2. 検査サマリー取得

GET /api/inspections/summary

指定したセクション＋日付の検査サマリーを取得します。

クエリパラメータ

section (必須), date (任意)

成功レスポンス (200 OK)

{
  "section": "alayer",
  "date": "2025-10-01",
  "total_count": 150,
  "pass_count": 145,
  "fail_count": 5,
  "pass_rate": 97
}

3. 検査ロット取得

GET /api/inspections/lots

ロット単位で検査結果を取得します。alayer セクションは four_k_sequences 配列を返します。

成功レスポンス (200 OK)

{
    "total_pages": 1,
    "lots": [
        {
            "lot_id": "LOLL-10001",
            "section": "alayer",
            "captured_at": "2025-10-01T17:47:53.427Z",
            "pass": false,
            "representative_image": "/api/images/afterTest/LOLL-10001_A-main01_A-1_A-1.png",
            "four_k_sequences": [
                { "4k_seq": "A-1", "status": "FAIL" },
                { "4k_seq": "A-2", "status": "PASS" }
            ]
        }
    ]
}

3.1 ロット詳細（ショット一覧: 4K / FHD）

GET /api/inspections/lots/{lot_id}/shots/4K

指定ロットの 4K ショットのみを返却します。 重要: DBの c4k_seq を API では 4k_seq というキーで返します。

成功レスポンス例

{
    "lot_id": "MAT-10001",
    "shots": [
        {
            "camera_id": "B-spring01",
            "4k_seq": "A-4",  // DB: c4k_seq
            "status": "PASS",
            "image_path": "MAT-10001_B-spring01-4k-02.jpg"
        }
    ]
}
GET /api/inspections/lots/{lot_id}/shots/FHD/{4k_seq}

指定ロット内で、特定の 4k_seq に紐づく FHD ショットのみを返却します。

成功レスポンス例

{
  "lot_id": "MAT-10001",
  "4k_seq": "A-4",
  "shots": [
    {
      "camera_id": "B-spring02",
      "fhd_seq": "D-5",
      "status": "PASS",
      "image_path": "MAT-10001_B-spring02-fhd-01.jpg"
    }
  ]
}
