日報管理API 設計書

対象ページ: 日報一覧ページ、日報入力ページ

このドキュメントは、従業員の日報を管理（CRUD）するためのAPI仕様を定義します。

これベースで開発ヨロ！

0. DB設計

日報データは、`employees` テーブルと `production_lines` テーブルを参照します。

daily_reports（日報テーブル）

CREATE TABLE daily_reports (
    report_id SERIAL PRIMARY KEY,                   -- 日報ID (自動採番)
    employee_id INT NOT NULL,                     -- 従業員ID (FK)
    line_id INT NOT NULL,                         -- ラインID (FK)
    report_date DATE NOT NULL,                      -- 報告日 (例: '2025-11-05')
    notes TEXT,                                   -- 備考・特記事項 (日記内容)
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, -- 作成日時
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, -- 更新日時

    FOREIGN KEY (employee_id) REFERENCES employees (employee_id),
    FOREIGN KEY (line_id) REFERENCES production_lines (line_id)
);

1. 日報一覧取得

GET /api/reports 優先度: S

日報のリストをまとめて取得するAPI。検索フィルターも使えるようにします。

クエリパラメータ (絞り込み検索)

パラメータ 説明 例
employee_name 従業員名（部分一致）で絞り込み。 ?employee_name=山田
line_name ライン名（部分一致）で絞り込み。 ?line_name=組立
start_date 期間検索（開始日）。 `report_date` がこの日以降。 ?start_date=2025-11-01
end_date 期間検索（終了日）。 `report_date` がこの日以前。 ?end_date=2025-11-30

成功レスポンス (200 OK)

※従業員名(employee_name)とライン名(line_name)は、API側でJOINして返します。

2. 特定の日報取得

GET /api/reports/{id} 優先度: S

IDを指定して、特定の日報1件の詳細情報をGETする。

成功レスポンス (200 OK)

{
    "report_id": 1,
    "employee_id": 12,
    "employee_name": "山田 太郎",
    "line_id": 1,
    "line_name": "組立ラインA",
    "report_date": "2025-11-05",
    "notes": "材料Aの供給が一時停止したため、午後から別作業。",
    "updated_at": "2025-11-05T17:30:00+09:00"
}

エラーレスポンス (404 Not Found)

{
    "message": "日報が見つかりません"
}

3. 日報追加

POST /api/reports 優先度: S

新しい日報を登録するためのAPI。データはJSONで送る。

リクエストボディ

{
    "employee_id": 12,
    "line_id": 1,
    "report_date": "2025-11-06",
    "notes": "本日は通常通り作業完了。"
}

成功レスポンス (201 Created)

登録が成功したら、IDが振られた日報データ（JOIN済み）が返ってくる。

{
    "report_id": 3,
    "employee_id": 12,
    "employee_name": "山田 太郎",
    "line_id": 1,
    "line_name": "組立ラインA",
    "report_date": "2025-11-06",
    "notes": "本日は通常通り作業完了。",
    "updated_at": "2025-11-06T17:31:00+09:00"
}

4. 日報情報更新

PUT /api/reports/{id} 優先度: A

IDで指定した日報の情報をまるっと更新する。

リクエストボディ

{
    "employee_id": 12,
    "line_id": 1,
    "report_date": "2025-11-06",
    "notes": "本日は通常通り作業完了。\n追記：B部品の在庫確認実施。"
}

成功レスポンス (200 OK)

{
    "report_id": 3,
    "employee_id": 12,
    "employee_name": "山田 太郎",
    "line_id": 1,
    "line_name": "組立ラインA",
    "report_date": "2025-11-06",
    "notes": "本日は通常通り作業完了。\n追記：B部品の在庫確認実施。",
    "updated_at": "2025-11-06T18:00:00+09:00"
}

5. 日報削除

DELETE /api/reports/{id} 優先度: A

IDで指定した日報を削除する。従業員APIと同様、物理削除で設計。

成功レスポンス (204 No Content)

削除が成功したら、ボディには何も返ってこない。
